Using OpenStreetMap Data to Create D&D Maps

Authored by: Orest Tuz

This map was created in QGIS v3.32.2

This map utilized US state and county data from Natural Earth Data for filtering what data would be imported through other tools.

OpenStreetMap Data was collected through the QuickOSM v2.2.3 plugin for QGIS.

Of note, These keys below were used within the QuickOSM tool to gather the data represented in these maps:

  • amenity
  • building
  • highway
  • shop

These keys were set to a Layer extent of Fayette County in Lexington, KY

That layer was produced by filtering the Natural Earth US County shapefile to just Fayette County.

The projected coordinate system used was EPSG:3089 [NAD83 / Kentucky Single Zone (ftUS)]

The scaling for each section of the map varied and is listed underneath each section

From Left to Right: (1:500), (1:2500), (1:50000)
